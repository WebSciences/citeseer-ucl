1. The org.archive.util.HTTPRecorder class

The intention is to allow Heritrix to handle all relevant protocols, not only http/https. The naming of HTTPRecorder indicates that it is only for HTTP. Hopefully it is not. We have not investigated it fully yet, but it might be that changing the name of the class is all there is to be done.
